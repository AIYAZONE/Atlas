# 项目立项与价值主张（Business Case）

> 文档定位：阐述 Jackery Atlas 的商业价值、ROI、资产沉淀与风险控制，供管理层决策
> 核心目标：以自研 Headless + 静态站（Vue 3 + Nuxt 3）替代昂贵/封闭的传统 CMS 与 Shopify 前端生态，实现降本增效与数据主权

## Why Now?（为什么现在做）
- 降本压力与效率诉求显性化：AEM/高阶 SaaS 成本高，自研可节省数十万/年授权支出
- 去平台化趋势：摆脱 Vendor Lock-in，将页面结构（JSON Schema）与前端代码（Vue 组件）沉淀为公司自有资产
- 技术栈现代化：从 Liquid 转向 Vue 3 + Nuxt 3，提升招聘吸引力与团队士气，降低 Skill Gap 风险

## ROI 分析（显性节省 vs 隐形投入）
- 显性节省
  - CMS 授权费：由高昂年费 → 自研与开源/低成本服务（Supabase/Vercel）
  - 前端插件月费：核心构建与编辑能力内化，停用 Page Builder 类第三方
  - 托管成本：SSG + CDN 带来极低计算成本与更佳性能/SEO
- 隐形投入
  - 重造轮子成本：评论/积分/推荐等功能在 Headless 下需重写前端 UI，需预留约 30% 额外工时
  - 运维成本：构建流水线、CDN、数据库备份与监控需规范化维护
  - 发布延迟：从“保存即生效”改为“保存→构建→发布”，存在 5–10 分钟延迟，需要运营预期管理与增量构建（ISR）

## 资产负债表（我们沉淀什么）
- 数据主权：页面 Schema（JSON）归档于自有数据库（如 Supabase），版本化与可审计
- 代码资产：前端组件库、渲染引擎（JSON → Vue）、适配层（Shopify Storefront API），可复用可扩展
- 工程能力：SSG 构建与性能优化体系，提升 SEO 与转化
- 人才与组织：现代栈（Vue 3/TypeScript/Tailwind/Pinia）提升招聘与留任，形成团队护城河

## 风险与退出机制
- 风险识别
  - 功能迁移难度：部分生态功能需自行实现或替换为具备 API 的 App
  - 运维复杂度：采用成熟 Serverless 组合（Vercel/AWS S3/CDN/Supabase）并制定值守与备份策略
  - 运营体验阵痛：通过 ISR 与预热策略降低构建延迟影响
- 退出机制
  - 架构保持与 Shopify 后端数据兼容，必要时可将域名切回原生主题；业务零中断，仅损失前端投入

## 合规与安全
- GDPR 与隐私合规：前端与数据采集策略可控，支持更严格的 Cookie/Consent 与数据最小化
- 版本与发布审计：内容变更与构建产物可追踪，支持回滚与灰度

## 结论与行动项
- 结论：Atlas 以明确的降本增效与资产沉淀为核心，同时以可控风险与明确退出机制保障决策安全
- 行动项：
  - 启动 MVP：首页/产品页/购物车闭环，避免多语言/复杂营销的早期扩张
  - 建立运营协作流程：制定发布窗口与预期管理，启用增量构建
  - 完成基础设施落地：CI/CD、CDN、数据库备份与监控
