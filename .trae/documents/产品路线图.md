# Jackery Atlas 产品路线图

## 愿景与定位

* 面向 Jackery 内部的可视化建站与静态发布基础设施

* 兼容 Shopify Theme JSON/Section/Block 数据结构，长期可扩展

## 阶段规划

* Phase 1（0–3 个月）：MVP (Supabase)

  * 单站点，首页/产品页/内容页

  * Section/Block JSON Schema 与渲染闭环

  * Shopify 产品读取，静态发布到 S3 + CloudFront

  * 打通购物车→Checkout→订单 Webhook 闭环（Supabase Edge Functions）

  * 快速落地：Supabase Auth + Database + Edge Functions

* Phase 2（3–6 个月）：扩展与演进 (NestJS)

  * 集合页、模板系统完善

  * 引入 NestJS 中间层，接管复杂业务逻辑

  * 多语言基础（路由/资源），SEO 自动化

  * 增量构建与缓存策略优化

* Phase 3（6–12 个月）：平台化

  * 主题迁移工具（Liquid → DSL/组件）

  * 权限与审计（RBAC/历史）

  * 组件市场与 A/B 实验能力

## 关键里程碑

* M1：MVP 验收（编辑→预览→发布→Checkout 跳转→订单接收）

* M2：模板/集合页上线，构建时长与稳定性达标

* M3：迁移工具可用于 1 个真实主题

* M4：权限/审计与增量构建完成

## KPI（示例）

* 构建时长（单页）：≤30s；稳定性：>99% 成功率

* 静态首屏：≤1s；Lighthouse ≥90

* 迁移效率：普通主题 1–2 人周

## 风险与依赖

* 对 Shopify Webhook 与 API 限额的依赖（需做缓存与退避）

* 渲染库与 Builder 的 schema 一致性治理

* 自建数据库运维复杂度（备份/故障切换/安全）
