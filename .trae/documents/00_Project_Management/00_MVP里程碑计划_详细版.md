# MVP 里程碑计划（详细版，8 周稳健）

> 目标：交付可搜索、可转化的单站点 MVP（首页/产品页/购物车闭环），并建立基础设施与运营协作机制
> 验收口径：Lighthouse ≥ 95（移动端）、页面可被搜索引擎抓取、Checkout 跳转 Shopify、构建后无 Node Runtime

## 周期概览
- 周 1–2：需求梳理与架构基线
- 周 3–4：Builder 与渲染引擎 MVP
- 周 5–6：Site Runtime 与 SEO/性能
- 周 7–8：购物车闭环、发布与监控

## 周 1–2：需求梳理与架构基线
- 产出
  - 页面信息架构（IA）与内容类型清单
  - Section/Block Schema 设计原则与示例集
  - 技术栈与 Monorepo 结构确认（Vue3/Nuxt3/Pinia/Tailwind/TS）
  - CI/CD 初始管线与分支策略
- 验收
  - INDEX 与路线图文档更新
  - 架构图（数据流/渲染路径/适配层）评审通过
- 风险与缓冲
  - 需求变更：冻结核心页面范围；以扩展点记录未来需求

## 周 3–4：Builder 与渲染引擎 MVP
- 产出
  - 可视化编辑器基本能力：新增/删除/排序 Section/Block、字段编辑、撤销/草稿/发布
  - JSON → Vue 渲染引擎：Schema First 渲染、插槽/变体支持
  - 与 Shopify Storefront API 的适配层草案（产品/集合读取）
- 验收
  - 非开发人员可完成首页配置与预览
  - Schema 变更可保存、版本化、可回滚
- 风险与缓冲
  - 拖拽与复杂交互：优先保障稳定与可维护，避免过度炫技

## 周 5–6：Site Runtime 与 SEO/性能
- 产出
  - Nuxt 3 SSG 站点运行时：文件路由、自动导入、静态构建
  - 性能优化：代码分割、资源预加载、图片优化、缓存策略
  - SEO：标题/描述/OG、sitemap、robots、结构化数据（产品页）
- 验收
  - Lighthouse（移动端）≥ 95
  - 搜索引擎可抓取（含产品页）
- 风险与缓冲
  - 图片与多媒体：统一媒体策略与 CDN 规则

## 周 7–8：购物车闭环、发布与监控
- 产出
  - 购物车与 Checkout 跳转 Shopify（保持后端数据完整）
  - 发布策略：保存→构建→发布流程、发布窗口与回滚
  - 监控/审计：错误告警、性能观测、版本与发布审计
  - 增量构建（ISR）与运营协作机制（预期管理）
- 验收
  - 购物车闭环可用、发布可回滚
  - 构建与监控链路稳定
- 风险与缓冲
  - 高峰发布：设置窗口与熔断策略；预热关键页面

## 角色与分工（模板）
- 产品/运营：定义内容结构与文案资产；确定发布节奏
- 前端/架构：Schema/渲染/运行时实现；性能与 SEO
- DevOps：CI/CD、CDN、备份与监控

## 里程碑验收清单（汇总）
- 非开发人员完成首页配置与发布
- 产品页数据来自 Shopify，结构化数据合规
- Lighthouse（移动端）≥ 95，关键页面可缓存与预热
- Checkout 正常跳转 Shopify
- 发布有审计与回滚；监控与告警生效
