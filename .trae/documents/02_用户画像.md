# Jackery Atlas 用户画像

## Admin（管理员）

* 目标：配置系统、管理用户与权限、设置站点/域名/CDN

* 痛点：多人协作变更难追踪；发布风险；合规与审计

* 行为：创建项目、邀请成员、设置默认 SEO/缓存策略、查看发布历史与告警

* 场景：上线活动页、设置多环境、查看构建失败原因

## Editor（编辑者）

* 目标：高效完成页面编辑与发布；所见即所得；数据绑定简单

* 痛点：动效/样式调试成本高；数据联动复杂；回滚困难

* 行为：拖拽 Section/Block、编辑属性、预览与发布、绑定产品/集合

* 场景：制作首页 Banner、产品列表、活动页；快速修复文案与图片

## Viewer（查看者/审核）

* 目标：审核发布内容，确保合规与质量

* 痛点：缺乏上下文与变更历史；无法标注与反馈

* 行为：查看预览链接、记录问题、触发重新构建或退回

* 场景：活动上线前审核、SEO 元数据检查、可访问性校验

## 开发者（内建扩展）

* 目标：扩展组件与 DSL；维护渲染/构建；对接外部 API

* 痛点：兼容性与版本治理；跨团队协作成本

* 行为：新增组件、优化渲染器、编写迁移工具、调优构建管线

* 场景：迁移 Shopify Theme、新增 Section 模板、优化增量构建
