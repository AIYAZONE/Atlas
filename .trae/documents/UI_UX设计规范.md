# Jackery Atlas UI/UX 设计规范

## 设计原则
- 编辑即结果：预览与生产渲染一致
- 配置驱动：基于 Schema 自动生成表单与 UI
- 非破坏性编辑：草稿与发布分离，可回滚
- 可访问性优先：WCAG 2.1 AA、语义化与键盘可达

## 信息架构
- 左：组件库（Section/Block 分类、搜索、拖拽）
- 中：画布（iframe 实时预览、选中高亮、标尺/网格可选）
- 右：属性面板（基于 DSL 自动生成，支持校验/提示/依赖显示）
- 顶部：工具栏（设备切换、保存、发布、预览链接）
- 全局：购物车入口（图标+徽标），Mini Cart 抽屉；Checkout 按钮显著

## 组件规范
- Section：可组合、可排序、可复制；暴露最小必要配置
- Block：字段类型与校验遵循 DSL（text/image/richtext/...）
- 可见性：page/type/abTest 等条件可选，默认简单可用
- 命名：统一英文 id + 人类可读 name；避免技术术语外露

## 交互规范
- 拖拽：拖拽手柄；放置位置高亮；支持键盘辅助移动
- 表单：即时验证；错误提示明确；支持重置与撤销（后续）
- 预览：设备切换（Desktop/Tablet/Mobile）；缩放与适配
- 发布：二次确认；展示构建任务进度与日志
- 购物车：加入成功反馈（Toast/Badge 增量）；Mini Cart 展示行项目与总计；空状态与错误提示
- 结算：Checkout 跳转前确认；失败重试与引导（返回购物车/联系客服）

## 视觉规范（建议）
- 色彩：深灰基础 + 品牌辅助色；状态色（成功/警告/错误）
- 字体：系统字体栈；字号 14/16/20；间距与栅格统一
- 图标：线性简洁；语义明确；可访问文本替代
- 动效：轻量化；避免影响性能与可达性

## 可访问性
- 语义化标签与 ARIA；焦点管理；Tab 顺序合理
- 对比度符合 AA；图片替代文本；可见的键盘焦点样式
- 组件表单具备标签/提示/错误状态
- 购物车与结算：按钮语义清晰；价格/数量可被屏幕阅读器正确朗读；键盘完整操作路径
