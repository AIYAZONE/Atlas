# 周会推进检查清单（MVP 8 周稳健）

> 用途：每周推进会议的标准化检查与验收，对应里程碑产出与风险缓冲

## 周会结构（30–45 分钟）

* 5’ 回顾目标：本周里程碑与验收项

* 15’ 产出核查：Demo/文档/数据与 [PR 列表](#术语解释)

* 10’ 风险与缓冲：新增风险、缓冲策略与负责人

* 5’ 行动闭环：下周任务、责任人与截止时间

## 通用检查项

* 本周交付是否满足验收口径（可演示与可回滚）

* 性能/SEO 指标是否达标或有提升趋势

* 监控/告警是否覆盖本周新增模块

* 文档是否更新（INDEX/路线图/架构图/操作手册）

## 周 1–2（需求与架构基线）检查

* [IA ](#术语解释)与内容类型清单完成并评审通过

* Section/Block Schema 原则与示例集提交

* Monorepo 结构与技术栈确定；CI/CD 初始管线可用

* 架构图（数据流/渲染路径/适配层）归档

## 周 3–4（Builder 与渲染 MVP）检查

* 编辑能力可用：新增/删除/排序、字段编辑、撤销/草稿/发布

* JSON→Vue 渲染引擎可用，插槽/变体支持验证

* Shopify Storefront API 适配层可读产品/集合

* 非开发人员完成首页配置与预览；Schema 版本化与回滚可用

## 周 5–6（Site Runtime 与 SEO/性能）检查

* Nuxt 3 SSG 运行时稳定：文件路由、自动导入、静态构建

* Lighthouse（移动端）≥ 95；资源优化（分割/预加载/图片/缓存）

* SEO 三件套：标题/描述/OG；sitemap/robots；产品结构化数据

* 媒体与 CDN 策略落地（规则与示例）

## 周 7–8（购物车闭环、发布与监控）检查

* 购物车与 Checkout 跳转 Shopify 可用

* 发布流程可回滚；发布窗口与灰度策略验证

* 监控/审计：错误告警、性能观测、版本与发布审计

* 增量构建（ISR）与运营协作机制生效（预期管理）

## 风险与缓冲记录（模板）

* 新增风险：描述 / 影响范围 / 触发条件

* 缓冲策略：技术/流程/人力措施

* 负责人与截止时间：Owner / Due

## 术语解释

* IA（Information Architecture，信息架构）

  * 指站点内容的结构与组织方式，包括页面类型、导航层级、URL 规划、分类/标签以及内容实体之间的关系

  * 在 Atlas 语境下通常落地为：站点地图（Sitemap）、内容类型与字段定义（如 Section/Block 的 Schema 原则）、页面模板与数据来源映射、导航菜单与 URL 规范

  * 作用：让可视化编辑（Builder）与渲染引擎基于一致的结构工作，减少返工，提升 SEO 与可维护性

* PR 列表（Pull Request 列表）

  * 指代码协作平台（如 GitHub）上的本周需审或已合并的变更集合，用于周会核查产出与进度

  * 通常包含：PR 标题、关联 Issue/里程碑、源/目标分支、变更范围概要（diff 大小、目录范围）、检查项状态（lint/测试/构建是否通过）、评审人与状态（open/merged/closed）、标签（feature/bugfix/chore）

  * 作用：把团队本周可交付的代码变更透明化，方便对照里程碑验收与风险讨论
